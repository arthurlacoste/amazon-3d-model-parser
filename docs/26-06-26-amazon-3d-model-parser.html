26-06-26 - Amazon 3D Model Parser

Contexte

Nettoyage et publication du parseur Amazon 3D. Nom public demandé: Amazon 3D Model Parser. Slug GitHub demandé: amazon-3d-model-parser.

Ce qui a été fait

  • Découpage de la logique PHP en classes courtes dans src/.
  • Validation URL Amazon centralisée dans AmazonProductUrl.
  • Support des shortlinks https://amzn.to/... avec validation de la destination finale Amazon.
  • Client cURL isolé dans AmazonPageClient.
  • Parsing des liens /view-3d et génération URL zip dans ModelLinkParser.
  • UI nettoyée dans index.php, CSS externe dans assets/css/home-light.css.
  • Tests PHPUnit ajoutés.
  • Test browserless Playwright ajouté dans tools/browserless-test.js.
  • README mis à jour.

Pourquoi

Ancien fichier unique trop couplé. Séparation nécessaire pour tester la validation, le parsing et le rendu sans dépendre du réseau Amazon.

Risques

  • Amazon peut changer HTML, headers nécessaires ou bloquer requêtes serveur.
  • Les shortlinks nécessitent deux requêtes: résolution, puis fetch de la page Amazon finale.
  • Le requestId dans URL viewer varie à chaque test.
  • Le fichier zip peut être inaccessible si Amazon invalide ou protège le modèle.

Axes de progression

  1. Ajouter cache court serveur pour limiter requêtes Amazon répétées.
  2. Ajouter téléchargement optionnel du zip avec validation taille/type.
  3. Ajouter extraction ASIN et normalisation URL produit avant requête.
  4. Ajouter rate limiting côté serveur.
  5. Ajouter CI GitHub Actions pour PHPUnit + browserless smoke test.
